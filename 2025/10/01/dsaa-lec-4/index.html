HeapSort

Introduction

The idea behind HeapSort is to find the largest element and move it to the end of the array. Then we repeat this process with remaining elements until the array is sorted. This idea is similar to the SelectionSort, but SelectionSort compares lots of elements to find the largest, while the HeapSort store knowledge gained from these comparisons for future use. This make the HeapSort very efficient on both runtime and storage, and it also sorts arrays in place. More specifically, we use a clever data structure, a heap to achieve this. ### Heap A heap is essentially an array imagined as being a binary tree. The elements are arranged row by row from left to right. Procedures Needed for the Algorithm

  1. Build-Max-Heap: produces a max-heap from an unordered array.
  2. Max-Heapify: maintains the max-heap property once the maximum (the root element) has been removed.
  3. HeapSort: sorts an array in place.
